# Project: o-horizons-site
# Корпоративный сайт Open Horizons — IT-аутсорсинг

## Контекст проекта
- Сайт компании Open Horizons (o-horizons.com)
- Текущее состояние: одностраничный лендинг на Next.js/JavaScript
- Цель: переход к полноценному корпоративному сайту
- Есть отдельный портал: portal.o-horizons.com
- GitHub: https://github.com/Alex-Matsko/o-horizons-site

## Целевая архитектура
- Next.js App Router (многостраничный)
- Sanity CMS для управления контентом
- Двуязычность: RU / EN
- Страницы: Home, /services, /services/[slug], /articles, /articles/[slug],
  /cases, /cases/[slug], /testimonials, /about, /contact

## Стек
- Next.js (App Router)
- TypeScript (если нет — перейти на TypeScript)
- Sanity CMS
- CSS Modules или Tailwind CSS
- Docker для локальной разработки

## Команды проекта
- Установка зависимостей: npm install
- Запуск dev: npm run dev
- Сборка: npm run build
- Запуск через Docker: docker compose up -d
- Остановка: docker compose down
- Sanity Studio: npx sanity dev (из папки /studio если есть)

## Правила работы
1. ВСЕГДА начинай с плана — покажи список шагов до того, как писать код
2. После изменений проверяй сборку: npm run build не должна давать ошибок
3. Коммиты в формате: feat/fix/refactor/chore: краткое описание на английском
4. Отвечай кратко — только суть, без лишних пояснений
5. Не меняй .env и секреты без явного запроса
6. Не генерируй тексты на страницах — вместо них пиши placeholder-контент
7. Не трогай Sanity schema без явного согласования
8. Если задача непонятна — уточни перед началом, не угадывай

## Критерии готовности задачи
- Код написан и работает
- npm run build проходит без ошибок
- Изменения закоммичены в ветку v2.0 и запушены на GitHub
- Если изменена страница — она отображается корректно в dev

## Запрещено
- Удалять существующие компоненты без явного разрешения
- Добавлять новые зависимости без обсуждения
- Делать push в main напрямую — рабочая ветка v2.0, в неё и пушим
- Генерировать финальные тексты для услуг, кейсов и статей — это делает владелец
